How it works

The problem

Businesses accumulate thousands of old quotes as Word and PDF files. When a repeat product needs quoting, staff dig through the archive by part number, find the most recent quote, and copy details into a new document by hand. This project automates that end to end.

The pipeline

  1. Messy archive. 25 sample quotes (2022–2026), each saved as both .docx and .pdf — 50 files. Three layouts (tables, paragraph blocks, business letters), six different labels for "part number", five price formats, four date formats, and deliberately missing fields.
  2. AI extraction. Each document's text is sent to Claude with a plain-English rules file (field synonyms, layout notes, "output null, never invent values"). The owner edits the rules file to change extraction behavior — no code changes. Structured JSON comes back via a forced tool call.
  3. SQLite. Quotes and line items land in a small relational database, indexed by part number. Indexing is resumable and cost about $0.60 for the whole archive.
  4. Template population. A search finds every occurrence of a part; the most recent quote date wins. Its price, name, and description are written into a clean Word template — the file you can download from this demo.

Validation methodology & honest results

The archive generator writes a ground-truth answer key alongside the documents. The extractor never sees it. After indexing, every extracted field is compared against the key: exact match for part numbers, prices (to the cent), quantities and dates; whitespace-normalized fuzzy match for names and descriptions. Missed line items and hallucinated ("spurious") line items both count against accuracy — silence and invention are the two ways extraction lies.
